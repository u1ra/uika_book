# 更新日志

本文记录面向使用者的主要变化。历史版本中的运行地址和构建产物哈希不再保留；当前部署方式以 [README.md](./README.md) 和仓库配置为准。

## v1.13 — 2026-07-18

本版修复目录规则页面新增、编辑自定义规则时不显示保存按钮的问题，并完善小屏弹窗布局与保存完成后的状态收尾。后端 API、数据库结构、目录解析逻辑和字段语义均未改变。

### 目录规则弹窗

- 修复“取消”和“保存规则”被放入 `DialogContent` 未实现的 `footer` 命名插槽，导致按钮 DOM 没有渲染的问题。
- 将操作区改为弹窗默认内容，新增规则和编辑规则继续复用同一套表单及保存入口。
- 弹窗最大高度限制在当前视口内，表单主体内容过长时独立滚动，底部操作区保持可见。
- 弹窗宽度增加视口边距约束，改善移动端和较窄窗口下的显示。
- 为表单内操作按钮明确设置 `type="button"`，避免未来改用原生表单后触发意外提交。

### 保存流程

- 新增统一的弹窗重置与关闭逻辑，成功创建或更新规则后清理编辑状态并刷新规则列表。
- 修复保存请求进行中因 `submitting` 关闭保护而阻止成功回调关闭弹窗的问题。
- 保存期间继续禁用取消和保存按钮，避免重复请求或中途丢失提交状态。

### 验证与部署

- `npm run build`：通过，包含 TypeScript 检查、Vite 生产构建以及 PWA 产物生成。
- `docker compose up -d --build frontend`：通过，前后端容器已重新创建并正常运行。
- 同源入口 `http://localhost:7234/` 返回 `200`，`/health` 返回正常状态。
- 已确认部署后的前端资源包含新版“保存规则”操作。
- Docker 数据卷未删除，现有书籍、目录规则和阅读进度数据保持不变。

## v1.11 — 2026-07-16

本版面向固定自托管的个人使用场景，将浏览器访问方式收敛为完全同源。前后端仍通过 REST API 解耦，但部署时只保留一个对外入口。

### 架构调整

- 删除运行时本地 / 远程后端切换、后端地址校验和后端状态提示。
- 删除 `BackendSwitchModal.vue` 与 `frontend/src/utils/backend.ts`。
- API、封面和健康检查统一使用 `/api`、`/media/covers`、`/health` 相对路径。
- 本地开发继续使用 Vite proxy，Docker 部署继续使用 Nginx proxy。
- 后端移除标准 `CORSMiddleware`、`ReflectCORSMiddleware` 与 `CORS_ORIGINS` 配置。
- Docker 不再向宿主机发布后端端口，只通过 Compose 网络向前端 Nginx 暴露 `8000`。
- 前端端口可通过 `FRONTEND_PORT` 修改，默认 `7234`。

### 登录兼容

- Token 恢复为单一固定存储键，不再按后端地址隔离。
- 升级时自动迁移当前 Origin 对应的旧同源 Token。
- 自动清理废弃的后端选择配置与切换提示，不会迁移其他远程后端的 Token。

### 影响

- 不修改后端业务接口、SQLite 表结构、章节加载方式和阅读进度语义。
- 浏览器不再支持从一个前端页面连接另一台后端。
- 域名或隧道部署必须把 `/api/`、`/media/covers/`、`/health` 与前端页面保持在同一个 Origin。
- curl 和服务端程序仍可直接访问 FastAPI；浏览器跨域直连不再受支持。

### 验证

- `npm run typecheck` 与 `npm run build`：通过，PWA 产物正常生成。
- 两个前端静态回归脚本：通过。
- `pytest -q`：`77 passed`；另有 1 条来自 FastAPI TestClient 依赖的弃用警告。
- `docker compose config` 与 `docker compose up -d --build`：通过。
- 容器实测：页面、登录路由和 `/health` 返回 `200`；`/api/books` 经同源代理返回预期 `401`；后端无宿主机端口映射。

## v1.10 — 2026-07-16

本版聚焦目录规则应用的可预期性、全站反馈一致性和移动端目录体验。后端 API、数据库结构以及阅读进度字段语义均未改变。

### 新增

- 书籍详情页新增“应用后预览”：
  - 选择目录规则后，使用现有 `POST /api/chapter-rules/test` 对当前书籍进行预检。
  - 展示预计章节总数和前 10 个章节标题。
  - `FULL_TEXT` 规则直接显示单章节“全文”。
  - 零匹配时明确提示实际应用将降级为“全文”单章节。
  - 使用请求序号忽略过期响应，快速切换规则时不会被旧结果覆盖。
- 新增全局 `AppNotificationDialog` 和通知队列：
  - 统一成功、错误、提示、警告及确认交互。
  - 删除书籍、分组和自定义规则不再调用浏览器原生 `confirm()`。
  - 危险操作支持独立标题、按钮文案和破坏性样式。

### 改进

- 移除 `vue-sonner`，应用反馈统一使用 Reka Dialog 与现有主题变量，明暗主题和移动端样式保持一致。
- 阅读页目录增加章节滑块快速跳转；长目录继续使用虚拟列表，打开目录时定位当前章节。
- 阅读页与书籍详情目录加宽滚动条，触屏设备更容易拖动。
- 移动端应用顶栏改为品牌、账号、导航的网格布局，隐藏冗长后端摘要。
- PWA Manifest 的 `display` 从 `fullscreen` 调整为 `standalone`，隐藏浏览器地址栏但保留系统状态区域。
- 远程后端配置增加 Mixed Content 预校验；HTTPS 页面会拒绝 HTTP 后端地址。
- HTTPS 请求失败时提供更明确的地址、服务和 SSL/TLS 排查提示。
- Docker 默认宿主机端口统一为前端 `7234`、后端 `8234`，Compose 默认固定 `DEBUG=false`。

### 行为说明

- 规则预览只读取测试结果，不会修改书籍目录；只有点击“重新解析目录”才会写入章节数据。
- 规则测试接口最多返回前 20 个匹配项，所以详情页最多展示其中前 10 个标题，但总章节数使用完整匹配计数。
- 本版不包含手势返回书架功能；相关试验改动已经回滚。

### 升级提示

- 前端依赖有变化，更新后需要重新构建镜像或重新执行 `npm install`。
- PWA 用户若仍加载旧资源，请先强制刷新；必要时注销旧 Service Worker 后重新打开。
- 当前 Compose 端口和 `DEBUG` 为固定值，修改根 `.env` 中同名端口或 `DEBUG` 不会覆盖它们；详情见 README。

### 验证

- `npm run typecheck`：通过。
- `npm run build`：通过，PWA Manifest、Service Worker 和 Workbox 产物正常生成。
- `node frontend/scripts/verify-ui-fixes.mjs`：通过。
- `node frontend/scripts/verify-reader-page-layout.mjs`：通过。
- `docker compose config`：通过。
- `pytest -q`：`72 passed, 2 failed`。失败项均位于 `backend/tests/test_deployment_config.py`，仍断言旧端口 `7000/21412`；当前实际配置为 `8234/7234`，测试需要后续同步。
